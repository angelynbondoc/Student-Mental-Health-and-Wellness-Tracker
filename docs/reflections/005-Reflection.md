# QA & Documentation Lead Reflection

## What decisions were hardest
The hardest decisions I faced as QA & Documentation Lead were setting documentation standards. I struggled to determine how detailed our documentation should be while balancing completeness with usability. This required multiple iterations and feedback sessions to find the right approach that worked for everyone. Choosing testing priorities was equally challenging because every feature seemed important to test thoroughly. I had to make tough calls about what to test first, focusing on critical user paths rather than trying to cover everything at once.

## How you handled a conflict or blocker
One conflict I remember clearly was when the development team said a feature was finished, but my testing showed it still had problems that would confuse users. Instead of going back and forth arguing, I wrote down exactly what a user would experience and shared that with everyone.

## What you would do differently
I would start documentation earlier next time. I waited too long to create standards, which meant we had messy documentation that needed fixing later. Starting from day one would prevent this extra work. I would ask more people for their opinions before making decisions. I made some choices without enough team input, which caused problems when people didn't like the changes. Having a small group help decide would work better.
