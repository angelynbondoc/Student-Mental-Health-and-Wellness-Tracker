# Test Cases: TC-007

**Project:** Student Mental Health & Wellness Tracker 

| **Field**    | **Description**    |
|---|---|
| **Test ID**    | TC-007    |
| **Feature**    | Mobile-Responsive Layout    |
| **Scenario**    | User verifies app layout on both mobile and web devices    |
| **Steps**    | 1. Open the app on a mobile phone.    |
|    | 2. Check all menu items (Home, Journal, Create, Habits, Resources, Inbox).    |
|    | 3. Verify the position of the menu bar on mobile.    |
|    | 4. Open the app in a browser on a computer.    |
|    | 5. Check all menu items again.    |
|    | 6. Verify the position of the menu bar on web.    |
| **Expected Result**   | All menu items are visible and functional on both mobile and web.<br>On mobile, the menu bar appears at the bottom of the screen.<br>On web, the menu bar appears on the left side of the screen.    |
| **Actual Result**    | All menu items are working correctly.<br>The menu bar appears at the bottom on mobile as expected.<br>The menu bar appears on the left side on web as expected.    |
